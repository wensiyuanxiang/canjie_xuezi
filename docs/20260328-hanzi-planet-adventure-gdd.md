# 《仓颉切字》游戏策划文档

## 一、基础信息

- **游戏名称**：仓颉切字
- **类型**：教育 + 休闲动作（切水果类）
- **平台**：移动端优先（iOS / Android），兼容 PC / Web
- **目标用户**：6-12 岁小学生（一纪对应低年级，二纪对应中年级，三纪对应高年级）
- **黑客松主题**：只能做一件事
- **输入方式**：触摸滑动 / 鼠标拖拽 / 摄像头手势识别（三选一，自动切换）

> 字从四方飞来，孩子挥指一划——切开正确的字让它归位，放过错误的字让它飞走。每学会一个字，世界就清晰一分。

## 三、核心设计理念

### 3.1 为什么是「切字」

切水果类游戏的注意力天然在飞出物上——玩家必须**看清每个字**才能决定切还是放。「切」是人类最本能的手势动作之一，6 岁孩子无需学习就能上手。比起左右移动挡板，划一刀的操控感更直接、更有力、反馈更爽。

更关键的是：切字支持**摄像头手势识别**——孩子可以在屏幕前挥手切字，全身参与，像体感游戏一样既学字又锻炼身体。这是其他玩法无法实现的独特卖点。

### 3.2 「只做一件事」的贯彻

| 层面 | 设计 |
|------|------|
| 输入 | 全程只有**划一刀**一个动作（手指/鼠标/手势，三选一） |
| 决策 | 全程只有**切 vs 不切**一个判断 |
| 能力触发 | 切开闪光字自动触发，不加按钮 |
| 系统 | 无卡槽、无技能树、无背包、无商店 |

### 3.3 教育理念

- **主动辨认 > 被动曝光**：孩子必须看清字才能决定切还是放过，字形辨认内嵌于玩法
- **玩中认形，停中认义**：游戏进行中训练字形辨认；关间结算时学习读音、组词、字源
- **无惩罚设计**：切错不扣分不扣命，漏切不罚关，只有正向反馈
- **间隔重复**：漏切的目标字在关底自动再出现；AI 追踪薄弱字并在后续关卡中增加频次
- **身体参与**：摄像头手势模式下，孩子挥手切字，全身运动，避免久坐

### 3.4 三种输入模式

| 模式 | 设备 | 操作 | 适用场景 |
|------|------|------|---------|
| 触摸滑动 | 手机/平板 | 手指在屏幕上划过字即切开 | 主要模式 |
| 鼠标拖拽 | PC/Mac | 按住鼠标拖拽划过字即切开 | 桌面备用 |
| 摄像头手势 | 任何有摄像头的设备 | 食指在空中划过，实时映射到屏幕轨迹 | 体感模式 |

三种模式在设置中切换，游戏逻辑完全一致——底层只有「划痕轨迹」这一个抽象输入。

**摄像头手势技术方案**：
- 使用 MediaPipe Hands（Google 开源，WASM 版本）
- 模型约 4MB，首次加载后本地缓存，**完全离线运行**
- 追踪食指指尖（landmark 8）的屏幕映射坐标
- 指尖移动速度超过阈值 = 判定为「切」，轨迹渲染为刀痕
- 无需服务器，无需网络，隐私安全

## 四、世界观

### 4.1 故事背景

仓颉造字时，降下了三场字雨：

- **第一场·名之雨**：给万物以名字——山有了形状，水有了轮廓，世界从虚无中显形。
- **第二场·述之雨**：给万物以动作与性情——风会吹、鸟会飞、花会开，世界活了过来。
- **第三场·意之雨**：将万物编织成故事与道理——山高水长、春去秋来，世界有了意义。

后来三场雨依次停歇。先是"意"消散——世界讲不通故事了；再是"声"沉默——万物静止不动了；最后"形"模糊——山不像山、水不像水，万物回归混沌。

字雨虽停，但那些字并未消失——它们变成了被墨壳包裹的「字石」，失去了光芒，在世界各处飞旋。仓颉留下了自己的「笔锋」——一道无形的刀刃。只要用笔锋划过字石，墨壳就会被切开，字从中释放，化为光点归位天穹。

孩子是新一代「切字人」，握着仓颉的笔锋，从最基本的"形"开始，一刀一刀划开字石，让世界重新看得见、动起来、有意义。切错了也没关系——干扰字石被切开后只是碎成墨屑散去，不伤不罚。

### 4.2 「字灵」——AI 伙伴

字灵是一个住在屏幕角落的小生物，有表情、会说话、偶尔犯傻。它不是老师，不是考官，是**陪伴者**。

- 游戏中：短句鼓励（不超过 5 个字）、切错时温柔纠正
- 过关后：用本关生字讲一句小故事（AI 生成）
- 开局时：帮孩子测评已有识字量

### 4.3 三纪结构

三场字雨对应三个大纪元，每纪赋予世界不同层次的能力，也对应儿童语言习得的自然顺序：

| 纪 | 名称 | 叙事 | 学什么 | 字数 | 对应年级 |
|----|------|------|--------|------|---------|
| 一 | **万物有形** | 第一场雨还了万物名字，山显轮廓、水现河道 | 名词为主，基础独体字 | ~800 | 一二年级 |
| 二 | **万物有声** | 第二场雨让万物动了起来，风吹鸟飞人笑 | 动词、形容词，合体字 | ~1200 | 三四年级 |
| 三 | **万物有意** | 第三场雨将万物编织成故事，世界有了道理 | 抽象词、文学词、成语 | ~1500 | 五六年级 |

同一个世界在三纪中各出现一次，但体验层层深化：

| 以「山野」为例 | 学什么 | 世界状态 |
|---------------|--------|---------|
| 一纪·有形 | 山、水、石、土、火（名词） | 山的轮廓出现，河流显形——**静态画面** |
| 二纪·有声 | 高、远、流、落、吹（动词/形容词） | 水开始流动，风开始吹，鸟开始飞——**动态画面** |
| 三纪·有意 | 峻、崎、蜿蜒、壮丽（复合/文学词） | 诗句浮现山间，场景有了叙事——**意境画面** |

### 4.4 八大世界

每纪包含八个主题世界，覆盖全部语义领域：

| 世界 | 主题 | 涵盖字类 | 一纪场景变化 | 二纪场景变化 | 三纪场景变化 |
|------|------|---------|-------------|-------------|-------------|
| 山野 | 自然地理 | 山水石土火河湖海岛… | 灰白轮廓 → 彩色山水 | 静态 → 风吹水流 | 出现诗句浮于山间 |
| 天穹 | 天象时令 | 日月星风雨雪春夏秋冬… | 灰暗天空 → 日月星辰 | 静态 → 四季流转 | 二十四节气走马灯 |
| 田园 | 动植物 | 牛马鸟鱼花草木虫… | 荒地 → 花鸟出现 | 静态 → 蝶飞鸟鸣 | 食物链/生态故事 |
| 家园 | 身体家庭 | 人口手目耳爸妈哥姐… | 空屋 → 家具人影 | 静态 → 家人活动 | 家庭故事场景 |
| 村落 | 生活器物 | 门窗桌椅衣食米刀车… | 空村 → 房屋器物 | 静态 → 烟火升腾 | 街巷叫卖声起 |
| 学堂 | 文教 | 文书笔纸字言读写… | 白纸 → 字迹浮现 | 静态 → 翻页书写 | 满卷诗文连篇 |
| 集市 | 社会职业 | 金工商医兵官王国… | 空街 → 招牌挂出 | 静态 → 人来人往 | 百业兴旺故事 |
| 星台 | 数理方位 | 一二三百千上下左右… | 暗星 → 星点亮起 | 静态 → 星座连线 | 星图叙事动画 |

每个世界通关后场景**永久变化**，孩子回到地图看到的是自己一刀一刀「切」出来的世界。三纪完成后，同一个世界经历了三次视觉升级：轮廓 → 动态 → 意境。

### 4.5 天书残页——功能词的归属

功能词（的、了、在、把、被、从、而、且、虽然、因为……）不属于任何物理世界。它们有自己的叙事身份：

> 仓颉造完万物之名后，写下最后一卷《天书》——用来把所有字连成句子。没有天书，即使认得"山"和"高"，也说不出"山很高"。天书在失语时碎裂，残页散落在各个世界之间的虚空里。

设计要点：

- **天书残页关**作为特殊关卡，出现在每两个世界之间的过渡地带
- 场景是星空虚空中飘浮的古卷碎片，字写在半透明的金色纸页上
- 每收集一批功能词，地图上两个世界之间的路被金色文字桥梁连通
- 玩法差异化：残页关以"组句模式"为主（填句补字、词语排序），而非单纯认字形
- 一纪约 7 组残页关（~50 字），二纪约 7 组（~80 字），三纪约 7 组（~170 字）

### 4.7 叙事—教学—关卡对齐框架（解决「有故事却无体系感」）

世界观若只写在文档里、关卡里只出现「认字形 / 听音」，玩家会感到「宏大设定」与「一关一关切字」脱节。以下框架要求：**每一关都能在策划表与 UI 文案上对应到「故事在推进什么」与「认知在练什么」**，形成可检查的三元组。

#### 4.7.1 主轴：三场字雨 = 三条认知主轴（全产品）

| 字雨（叙事层） | 世界状态（玩家可见） | 认知主轴（教学层） | 关卡模式优先序 |
|----------------|----------------------|--------------------|----------------|
| 第一场·名之雨 | 轮廓与名目回归：灰白 → 彩色静态 | **形**：独体、部首、近形辨析 | 认字形 → 听音 → 偏旁 → BOSS 拆合 |
| 第二场·述之雨 | 动态与性情回归：静 → 动 | **用**：动词、形容词、搭配与语序 | 听音 → 组词连切 → 反义 → 天气加压 |
| 第三场·意之雨 | 意义与篇章回归：景 → 句 → 境 | **义**：语境、成语、抽象与文学词 | 填句 → 反义/近义 → BOSS → 混合复习 |

**执行规则**：同一世界在三纪中复用场景资产，但**主轴不同**——一纪关卡的结算文案强调「世界又有了名字」；二纪强调「世界又会动了」；三纪强调「世界又会讲故事了」。避免三纪都用同一套话术。

#### 4.7.2 单世界「十章」标准幕结构（系统感来源）

每个世界 10 关不是随意堆模式，而是固定五幕 × 每幕 2 关的节奏模板（可因字数微调，但**幕名与意图不变**）：

| 幕 | 关位（典型） | 叙事节拍（玩家可理解的一句话） | 教学意图 | 玩法侧重 |
|----|----------------|----------------------------------|----------|----------|
| 第一幕·显形 | 1-2 | 「混沌里先认出第一个名字」 | 新字初见、建立目标字概念 | 认字形为主，干扰逐步出现 |
| 第二幕·辨声 | 3-4 | 「名字不仅有形，还有声」 | 音形绑定、巩固已学 | 听音切字 + 认字形混合 |
| 第三幕·归类与联结 | 5-7 | 「字与字是一家人 / 能组成词」 | 结构（偏旁）或词语顺序 | 偏旁或组词；中点 BOSS 强化构造 |
| 第四幕·挑战 | 8-9 | 「相似与混合里仍能认准」 | 近形、混合模式、天气加压 | 混合复习、提高信息密度 |
| 第五幕·定型 | 10 | 「大块字石裂开，世界这一角定型」 | 合体字逻辑、本章收束 | BOSS；过关后场景永久升阶 |

BOSS 放在每幕交界或第五幕是**叙事句**：「大块墨壳 = 世界一块区域仍被混沌锁住，拆开它，这一角才归位。」

#### 4.7.3 每关必须落地的「三件套」（策划与 UI 同构）

策划表与游戏内**战前预览**共用同一信息结构，避免「关是数据、故事是另一套」：

1. **叙事钩子**（1 句，≤20 字）：本关在「救回哪一层世界」——例如一纪山野「让山从墨里浮出轮廓」。
2. **认知目标**（1 条）：本关主练的能力（辨认形 / 音形对应 / 结构 / 语境等）。
3. **世界锚点**（1 条）：过关后场景**具体**变化（与 7.4 节条目一一对应，可勾选实现）。

评审检查：任意一关若写不出以上三条，该关要么合并进相邻关，要么拆成两关以降低认知负荷。

#### 4.7.4 地图与进度条：把「宏大」收成「可见系统」

即使 MVP 只有单世界少量关，也应在地图上呈现**与故事同构的进度语言**（不必做完整八世界）：

- **墨壳覆盖率 / 轮廓收集度**：随已释放字数下降或上升，让玩家感到在「切回一场雨」，而非孤立小游戏。
- **当前纪元标题常驻**：例如「一纪·万物有形 — 山野」；切到二纪同场景时副标题改为「二纪·万物有声 — 山野」，强化「同一星球、不同历史阶段」。
- **世界节点三段式图标**：未解锁 / 进行中（墨壳缠绕）/ 已定型（彩色立绘），与 4.4 节「通关后永久变化」一致。

## 五、核心玩法

### 5.1 基础规则（切字）

字石从屏幕底部和两侧**向上抛出**，沿抛物线飞行，到达顶点后落下并飞出屏幕。玩家用手指/鼠标/手势划出「刀痕」，划过字石即切开。

- 字石被切开后**一分为二**，两半沿切割方向飞开，同时释放光粒子特效
- 每颗字石在空中停留约 **2-3 秒**（从出现到飞出），窗口期内可切
- 同屏最多飞出 **3-5 颗**字石，从不同方向、不同时间抛出
- **一刀可切多字**：一次划痕轨迹经过多颗字石，全部切开（连切），combo 连续累加
- 字石未被切到 = 安全飞走，不扣分不扣命

**判定规则**：

- **切对**（切开目标字石）：字从墨壳中释放，化为金色光点飞向天穹 → combo +1 → 音效 + 粒子爆炸
- **切错**（切开干扰字石）：字石碎成灰色墨屑散去 → 字灵温柔提示「这个不是哦」→ combo 归零，不扣分不扣命
- **放过**（不切任何字石）：字石安全飞走，不影响任何数值
- **漏切**（目标字石飞走未切）：不惩罚，关底会再出现一次

**核心操作快感**：
- 一刀三连切（同时切开三颗目标字）→ 屏幕震动 + 慢动作回放 + 三倍光粒子
- 精准切割（刚好划过字的中心线）→ 额外完美切割特效
- 手势模式下，挥手的体感 + 视觉爆炸反馈 = 沉浸式体验

### 5.2 七种关卡模式

#### 模式 1：认字形——「切对的」

屏幕上方显示目标字（如「山」），字石从各方向飞来，切开目标字石、放过其余。

认知重点：字形辨认。用于新字初见。

#### 模式 2：听音切字——「听声切字」

字灵念一个读音（如「shān」），不显示目标字形，孩子从飞来的字石中找到对应字切开。

认知重点：音形对应。比模式 1 高一级。

#### 模式 3：偏旁归类——「一家人」

屏幕提示一个偏旁（如「氵」），切开所有含该偏旁的字石，放过其余。

认知重点：结构分析、偏旁意识。

#### 模式 4：组词连切——「先后有序」

屏幕提示词语（如「春天」），必须按顺序先切「春」再切「天」。切错顺序不罚分但该词不计。

认知重点：词语意识、字序。

#### 模式 5：反义配对——「找反面」

屏幕显示一个字（如「大」），切开其反义字（「小」）的字石，放过其余。

认知重点：语义理解。

#### 模式 6：填句补字——「缺了谁」

屏幕显示带空格的句子（如「太___出来了」），切开正确的字石（「阳」）。

认知重点：语境理解。最高阶模式。

#### 模式 7：BOSS 拆字——「拆字大战」

一个合体字（如「休」）作为巨大 BOSS 字石悬浮在屏幕中央，外裹多层墨壳。孩子需要切开飞来的组件字石（「亻」和「木」），被切开的组件飞向 BOSS 吸附拼合。每吸附一个组件，BOSS 的墨壳裂开一层。全部组件就位后 BOSS 墨壳碎裂，合体字释放光芒。

认知重点：汉字构造逻辑。每 5 关出现一次。

### 5.3 天气系统（飞行变量）

不同关卡有不同天气，影响字石的**飞行轨迹和可切窗口**，增加操作变量和视觉新鲜感：

| 天气 | 飞行效果 | 切割影响 | 适用场景 |
|------|---------|---------|---------|
| 晴天 | 标准抛物线 | 标准切割窗口 | 默认 |
| 微风 | 字石轻微飘移偏转 | 需要预判轨迹提前出刀 | 增加瞄准难度 |
| 大雨 | 字石飞得更低更快 | 可切窗口缩短，需快速反应 | 精准挑战 |
| 飘雪 | 字石缓慢飘浮，滞空时间长 | 同屏字石数量翻倍，辨认压力大 | 信息密度挑战 |
| 旋风 | 字石自转飞行，字形旋转 | 必须在字转到正面时切割 | 辨认+时机双重挑战 |

天气和世界主题绑定：山野多风雨、田园是晴天、天穹大雨、星台飘雪。二纪起解锁旋风天气。

## 六、难度体系

### 6.1 九个难度旋钮

| 旋钮 | 低 | 中 | 高 |
|------|-----|-----|-----|
| 飞行速度 | 慢（滞空 3s） | 中（滞空 2s） | 快（滞空 1.2s） |
| 同屏字数 | 1 个 | 2-3 个 | 4-5 个 |
| 目标/干扰比 | 全目标 | 1:1 | 1:3 |
| 干扰字相似度 | 完全不像 | 有点像 | 极像（己/已/巳） |
| 新字数量 | 1 个新字 | 2 个新字 | 0（纯复习） |
| 认知模式 | 认字形 | 听音/偏旁 | 组词/填句 |
| 字石大小 | 大（易切） | 中 | 小（需精准） |
| 抛出方向 | 仅从底部 | 底部+两侧 | 四面八方 |
| 飞行轨迹 | 标准抛物线 | 带弧度 | 不规则曲线 |

**核心原则**：每关只调 1-2 个旋钮。新字多的关飞行一定慢；飞行快的关一定只出已学字。操控旋钮（字石大小、抛出方向、轨迹）从二纪起才调高。

### 6.3 关键数字

| 指标 | 值 | 原因 |
|------|-----|------|
| 单关时长 | 45-90 秒 | 6 岁注意力极限约 2 分钟 |
| 单关抛出总数 | 15-35 个 | 太少无趣，太多疲劳 |
| 单关新字上限 | 2 个 | 超过 2 个记忆负担陡增 |
| 同屏字数上限 | 5 个 | 再多视觉混乱 |
| 过关最低切对率 | ~50% | 让绝大多数孩子能过 |
| BOSS 间隔 | 每 5 关 | 节奏：学-学-练-学-BOSS |
| 每世界关卡数 | 10 关 | 约 20 分钟一个世界 |

### 6.4 AI 动态微调（后台无感）

**速度微调**：
- 连续 3 关切对率 > 90% → 下一关飞行速度 +10%
- 连续 2 关切对率 < 50% → 下一关飞行速度 -15%

**干扰字选择**：
- 追踪切错最多的字对（如总把「出」当「山」切）
- 后续关卡刻意多投放该字对
- 某字连续 3 次切对 → 降低出现频率，把机会让给薄弱字

## 七、奖励与解锁体系

### 7.1 即时反馈（每次切开字石）

- 目标字石被切开 → 一分为二飞散 → 字从中释放化为金色光点飞向天穹 → 切割音效 + 光粒子爆炸
- 一刀多切（一划切开多颗字石）→ 额外爆裂特效 + 连切音效递进
- 精准切割（划过字石中心线）→ 完美切割彩虹特效
- combo 3：刀痕变为金色
- combo 5：背景音乐加一层鼓点 + 刀痕拖尾变长
- combo 10：全屏慢动作 2 秒 + 刀痕变为火焰刀痕
- combo 断了（切错干扰字）：特效消失，不惩罚
- **不显示数字分数**，用视觉冲击代替，避免考试联想

### 7.2 过关评价（三档主题化）

不用星星，用世界主题符号：

| 档位 | 条件 | 奖励 |
|------|------|------|
| 种子 | 完成过关条件 | 解锁下一关 + 新字入库 |
| 幼苗 | 切对率 >= 70% | 额外能力碎片 x1 |
| 大树 | 切对率 >= 90% 且 0 切错 | 额外能力碎片 x2 + 隐藏字线索 |

每个世界用不同符号（山野：种子/幼苗/大树；村落：小屋/楼房/城堡；学堂：纸/书/卷轴；天穹：星星/月亮/太阳；田园：芽/花/果）。种子档即可推进，不逼孩子刷满。

### 7.3 字变能力

学会特定词根字后解锁能力。能力以**闪光字石**的形式在后续关卡中低概率出现，切开即触发（不加按钮）。

| 词根 | 能力名 | 效果 |
|------|--------|------|
| 火 | 焚字 | 切开后火焰扩散，烧掉空中所有干扰字石 |
| 水 | 墨流 | 刀痕变为水流，经过的所有目标字石自动切开（5 秒） |
| 风 | 风息 | 全屏暂停，所有字石悬停空中 3 秒，从容辨认再切 |
| 大 | 巨化 | 所有字石放大 2 倍，5 秒内超容易切中 |
| 日 | 照明 | 所有目标字石高亮金色发光 10 秒，一眼可辨 |
| 刀 | 剑气 | 下一刀自动延伸为全屏横扫，切开一切目标字石 |

能力和字义直接关联，孩子记住的不是抽象符号，而是「火 = 能烧东西」「刀 = 横扫千军」的语义绑定。

### 7.4 世界场景变化（最大奖励）

每学会一个字，背景立刻产生对应变化。三纪的变化层层叠加：

**一纪·有形**（轮廓出现）：
- 学会「山」→ 背景出现山的灰色轮廓
- 学会「水」→ 山脚出现河流线条
- 通关：灰白轮廓永久变为彩色静态画面

**二纪·有声**（画面动起来）：
- 学会「流」→ 河水开始流动
- 学会「飞」→ 鸟开始振翅
- 通关：整个世界从静态变为动态，风吹草动

**三纪·有意**（意境浮现）：
- 学会「高」→ 山间浮现"山高"二字
- 学会「壮丽」→ 背景出现诗句
- 通关：世界成为一幅有故事的活画卷

孩子回到地图看到的是**自己一刀一刀「切」出来的世界**，而且每一纪回来都能看到世界又进化了一层。

### 7.5 隐藏内容与彩蛋

- 极低概率飞出**金色字石**（课标外趣味字，如「龘」「犇」），切开后字灵惊讶地科普
- 特定组合触发**成语彩蛋**（一关内切开山+高+水+长 → 弹出「山高水长」动画）
- 字灵图鉴：收集的字可以两两组合，字灵用 AI 造一个句子

### 7.6 解锁机制

**关卡解锁**：线性解锁，通关第 N 关即开放第 N+1 关。不设星级门槛，6-9 岁孩子被卡关会直接弃游。

**已认识的字跳过**：
- 开局 AI 测评或家长手动标记已认识的字
- 已认识字对应的教学关可一键跳过，直接获得字入库 + 能力碎片
- BOSS 关和新模式关不可跳过（玩法体验价值）
- 跳过后世界场景仍然变化

**世界解锁**（一纪内）：

| 世界 | 解锁条件 |
|------|---------|
| 山野 | 默认开放 |
| 天穹 | 通关山野 |
| 田园 | 通关天穹 |
| 家园 | 通关田园 + 字库 >= 20 字 |
| 村落 | 通关家园 |
| 学堂 | 通关村落 + 字库 >= 50 字 |
| 集市 | 通关学堂 + 完成 3 次组词接龙 |
| 星台 | 通关集市 + 字库 >= 80 字 |

**纪元解锁**：

| 纪 | 解锁条件 |
|----|---------|
| 一纪·万物有形 | 默认开放 |
| 二纪·万物有声 | 一纪通关 4 个世界 + 字库 >= 300 字 |
| 三纪·万物有意 | 二纪通关 4 个世界 + 字库 >= 1200 字 |

二纪、三纪的八大世界沿用同样的线性解锁顺序，但不要求一纪全部通关才能进入二纪——通关一纪半数世界即可，降低卡关风险。

## 八、AI 系统设计

### 8.1 AI 的定位

AI 是「字灵」的大脑。对孩子来说，字灵是一个有性格的小伙伴；对产品来说，AI 承担四个职责。

### 8.2 实时旁白（游戏进行中）

- 切对：随机短句（「好刀法！」「就是它！」「漂亮！」）
- 连续切对：「厉害！」「三连斩！」
- 切错：「这个是 X，不是我们要找的哦」（一句话纠正，不暂停）
- 漏切：不说话，字灵做一个「哎」的表情

规则：进行中反馈**不超过 5 个字、不暂停游戏、不遮挡视线**。

### 8.3 关间讲解（核心教学时刻）

过关后 10-15 秒结算画面：

1. 展示本关新学的字（字卡 + 读音播报）
2. AI 生成一句话故事串联本关生字（如「山上有木，木多成林，下起大雨，雨落成水」）
3. 可选：点击字卡听详细讲解（字源、组词、造句）
4. 孩子可跳过，不强制

这是 AI 真正发力的地方：用大模型生成包含本关生字的短故事，每次不同，有新鲜感。技术上只是一次简单的 prompt 调用。

### 8.4 自适应难度（后台无感）

见第六章 6.4 节。

### 8.5 个性化复习（字灵日记）

每次打开游戏，字灵说：「上次你学了 X 个字，我们先复习 3 个吧？」进入 30 秒快速复习关（只含已学字，不出新字）。

AI 根据艾宾浩斯遗忘曲线选择复习字：即将遗忘的优先出现。

### 8.6 AI 安全边界

- 汉字读音、组词从**本地词表校验**，不允许模型胡编
- 生成的故事经过**内容过滤**，仅允许儿童友好内容
- 无社交、无聊天、无开放对话，AI 只在封闭场景内生成
- 所有数据本地存储，不上传个人信息

## 九、美术风格

### 9.1 整体方向

Q 版卡通、柔和治愈的扁平矢量风。低饱和度马卡龙配色，无尖锐棱角、无刺眼色彩。

### 9.2 关键设计原则

- **字要大**：字石上的字号不低于屏幕高度的 8%，粗体，高对比
- **背景要静**：游戏进行中背景是低对比的灰白/浅色，不和字抢注意力
- **色彩有语义**：目标字暖色调（橙/金），干扰字冷色调（灰蓝），能力字闪光
- **字灵要小**：占屏幕角落不超过 10%，不挡视线
- **字体**：选用覆盖 3500 常用字的开源教育字体，如思源黑体 Bold

### 9.3 世界地图 UI

地图为纵向三层结构，对应三纪（底部一纪 → 顶部三纪），像一棵生长的大树：根基是万物之形，树干是动作与声音，树冠是意义与故事。

每纪内八大世界沿水平路径排列，世界之间有「天书残页」节点作为桥梁。未解锁纪元整体朦胧；已解锁世界明亮、未解锁世界灰白。关卡节点为圆形光点，已通关为金色，当前关卡呼吸动效，天书残页节点为金色卷轴图标。

## 十、音效与音乐

### 10.1 音效设计

| 事件 | 音效风格 |
|------|---------|
| 切开目标字石 | 清脆切割声「嚓」+ 光粒子爆发音 + 字读音（TTS） |

| 过关 | 欢快短旋律 |

- 通关后为完整版（仪式感）

### 13.3 MVP 如何仍显得「有世界观、有体系」（与 4.7 节配套）

黑客松关卡少，若不做显式绑定，极易退化成「切字小游戏」。MVP 建议固定下列最小集，与完整版同一套语言，便于日后扩展：

| 要素 | MVP 落地 | 目的 |
|------|----------|------|
| 纪元与世界标题 | 开局 10-15 秒：字灵 3 句内讲完「第一场名之雨停歇 → 山野被墨壳锁住」；HUD 常驻「一纪·万物有形 · 山野」 | 把单世界放进三纪叙事，而非无名关卡列表 |
| 进度条 | 地图或关前列「墨壳消散 / 轮廓回归」百分比（按已释放目标字数或关进度） | 把「救世界」做成可见系统 |
| 每关三件套 | 战前 1 句叙事钩子 + 1 条认知目标 + 过关 1 条世界锚点（见 6.2 表） | 教学目的与故事同屏可见 |
| 幕结构 | 6 关可压缩为：显形(2) → 辨声(1) → BOSS/联结(1) → 挑战或复习(1) → 终 BOSS(1)，幕名在选关界面用小标签展示 | 玩家感到「章节感」而非 1-6 数字 |
| 终局仪式 | 第 6 关通过后播放 5-8 秒「山野由灰变彩」与字灵一句收束 | 与完整版「世界永久变化」同一体验锚点 |

**MVP 6 关与完整 10 关的对应关系（策划引用）**：完整版关 1-2-3-4-5-10 可映射为 MVP 关 1-2-3-4-5-6（中间偏旁、组词、混合复习在 MVP 可合并进第 5 关 BOSS 前后半段或省略，但须在文档与路演中说明「完整版将展开为十章五幕」）。

### 13.4 答辩话术

> 「仓颉切字」是一款面向 6-12 岁的识字游戏，覆盖小学全部 3500 常用字。故事说仓颉当年降下三场字雨，给了世界形、声、意。后来字雨停了，世界失语——字化为墨壳包裹的字石飞旋于天地间。仓颉留下了自己的笔锋，孩子是新一代「切字人」，全程只做一件事——用手指/手势划一刀，切开正确的字石让字归位。你必须看清每个字才能决定切还是放过，玩法本身就是在认字。它还支持摄像头手势识别——孩子挥手切字，全身参与，既学字又锻炼身体。每学一个字，世界就清晰一分——山出现轮廓、水开始流动、诗句浮现山间。三纪的旅程，让孩子亲手把世界从混沌中「切」出来。今天演示的是一纪·万物有形第一世界「山野」的 6 关完整流程。

## 十四、风险与应对

| 风险 | 应对 |
|------|------|
| 切与放重复感 | 7 种模式轮换 + 天气变量 + 能力系统 + 一刀多切连切快感 |
| 孩子不看字乱切 | 切错干扰字 combo 归零；形近字迫使辨认再下刀 |
| 摄像头手势不准 | 提供灵敏度调节；低端设备自动降级为触摸模式 |
| 摄像头隐私顾虑 | 所有手势处理纯本地运行，不传图像，不录像，不上传 |
| AI 生成不可控内容 | 本地词表校验 + 内容过滤 + 封闭场景 |
| 3500 字内容量巨大 | 三纪 × 八世界 + 天书残页结构化分配；数据驱动，字库 JSON 可持续扩展 |
| 教育效果难量化 | 内置字灵图鉴统计；后续版本加家长报告 |
| 字体缺字 | 使用覆盖 GB2312 的开源字体，启动时校验 |
